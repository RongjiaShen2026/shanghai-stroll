
# 数据结构课程的核心内容

基础概念

1. 什么是数据结构 vs 算法
2. 抽象数据类型（ADT）：先定义"这个东西能做什么"，再实现"怎么做"
3. 时间复杂度和空间复杂度：大 O 记号（O(1)、O(log n)、O(n)、O(n²)）
4. 递归思维：一个函数调用自己解决子问题


## 1. 线性数据结构
按"元素一个接一个排列"的方式组织数据。

|结构|中文|关键操作|典型场景|
|---|---|---|---|
|Array / List|数组 / 列表|按下标访问 O(1)|Python 的 list||Linked List|链表|中间插入 O(1)|不用记，知道概念即可|
|Stack|栈|LIFO（后进先出）|撤销操作、函数调用栈|
|Queue|队列|FIFO（先进先出）|排队任务、BFS|
|Deque|双端队列|两头都能进出|滑动窗口|


## 2. 哈希结构

1. Hash Table 哈希表：通过"哈希函数"把键变成数组下标
2. Hash Map / Dictionary：Python 的 dict
3. Hash Set：Python 的 set
4. 冲突处理：两个键算出同一个下标怎么办（链地址法、开放寻址法）

**关键洞察：牺牲一点内存，换接近 O(1) 的查找速度。**


## 3. 树结构

数据有"父子层次关系"时用。

1. 二叉树 Binary Tree：每个节点最多两个子节点
2. 二叉搜索树 BST：左小右大，查找 O(log n)
3. 平衡树：AVL 树、红黑树（防止 BST 变成单链）
4. 堆 Heap：优先队列的实现 ★  Dijkstra algorithm用的 heapq 就是堆.
5. Trie 字典树：前缀匹配，输入法、自动补全

## 4、图结构 ★

1. 图的表示：邻接矩阵 vs 邻接表（你用的就是邻接表）
2. 有向图 vs 无向图：你的 Digraph vs Graph
3. 加权图：每条边带数字（你的 WeightedDigraph，权重是米数）
4. 图的遍历：
      BFS 广度优先搜索 → 用队列
      DFS 深度优先搜索 → 用栈或递归

5. 最短路径算法：
       Dijkstra → 非负权重
       Bellman-Ford → 允许负权重
       Floyd-Warshall → 所有节点对之间

6. 最小生成树：Prim、Kruskal
7. 拓扑排序：处理"依赖关系"（比如课程先修关系）


## 5. 排序算法
把数据按顺序排列，是经典话题。

|算法|复杂度|特点|
|---|---|---|
|冒泡 / 插入 / 选择排序|O(n²)|简单但慢，教学用|
|归并排序|O(n log n)|稳定，分治思想|
|快速排序|O(n log n) 平均|实战最快|
|堆排序|O(n log n)|用堆这个数据结构|


## 6. 搜索算法

1. 线性搜索：一个一个看 O(n)
2. 二分搜索：有序数据里折半找 O(log n)

## 7. 进阶（有些课会教，有些不会）

1. 动态规划 DP：把大问题拆成重叠子问题
2. 贪心算法：每一步选当前最优（Dijkstra 就是贪心！）
3. 分治：分而治之（归并排序、快排）
4. 回溯：DFS + 剪枝（八皇后、数独）


# dictionary
```python
d = {'name': '汾阳路', 'length': 87.4, 'highway': 'residential'}

# 1. 拿所有 key
d.keys()
# dict_keys(['name', 'length', 'highway'])

# 2. 拿所有 value
d.values()
# dict_values(['汾阳路', 87.4, 'residential'])

# 3. 同时拿 key 和 value（最常用）
d.items()
# dict_items([('name', '汾阳路'), ('length', 87.4), ('highway', 'residential')])

# 4. 拿单个 value
d['name']           # '汾阳路'    — key 不存在会报错
d.get('name')       # '汾阳路'    — key 不存在返回 None
d.get('name', '无名')  # '汾阳路' — 不存在时返回 '无名'
```

最常见的遍历写法：

```python
# 只看 key
for key in d:                 # 等价于 for key in d.keys()
    print(key)

# 只看 value
for value in d.values():
    print(value)

# 同时拿 key 和 value（用得最多）
for key, value in d.items():
    print(f"{key} = {value}")
```
**for key, value in d.items(): 这个写法你会在真实代码里见得非常多。**

## dict 的内部实现叫 哈希表（hash table）。这是计算机科学里最经典的数据结构之一。

```python
# 用 list 找东西 —— 必须一个一个看
my_list = ['Alice', 'Bob', 'Carol', ..., 'Zack']
'Mike' in my_list           # 平均要看一半 → O(n)，慢

# 用 dict 找东西 —— 几乎瞬间
my_dict = {'Alice': 1, 'Bob': 2, ..., 'Zack': 26}
'Mike' in my_dict           # 一步到位 → O(1)，快
```
dict 的查找快到几乎和列表大小无关——10 个元素和 1000 万个元素，查找速度差不多。

## 总结：dict 的核心能力

|操作|写法|速度|
|---|---|---|
|加键值对|d[key] = value|极快 O(1)|
|取值|d[key] 或 d.get(key)|极快 O(1)|
|检查键存在|key in d|极快 O(1)|
|删除|del d[key]|极快 O(1)|
|遍历|for k, v in d.items():|O(n)|

# dictionary

```python
d = {'name': '汾阳路', 'length': 87.4}

d.keys()        # dict_keys([...])
d.values()      # dict_values([...])
d.items()       # dict_items([(k, v), ...])
d['name']      # crashes if missing
d.get('name')  # returns None if missing
d.get('name', 'default')  # returns 'default' if missing

# Most common iteration
for k, v in d.items():
    print(f"{k} = {v}")
```

### dict is a hash table
Hash tables are a classic CS data structure. Their key feature: lookups are roughly constant time regardless of size — 10 items vs 10 million items, similar speed.

```python
'Mike' in my_list   # O(n) — has to scan
'Mike' in my_dict   # O(1) — direct jump
```

Core dict operations (all O(1))

```python
d[k] = v        # add/update
d[k]            # get (crashes if missing)
d.get(k)        # get (returns None if missing)
k in d          # check existence
del d[k]        # delete
for k, v in d.items():   # iterate
```
